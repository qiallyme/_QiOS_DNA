# QiAccess Responsibilities

QiAccess is responsible for:

- Providing the main portal and dashboard.
- Opening real tools and services quickly.
- Presenting the seven-root navigation contract.
- Surfacing Capture as a fast path.
- Pointing Knowledge to canonical docs and references.
- Showing System visibility without becoming the system layer itself.
- Keeping Memory and Insights clearly marked when they are placeholders.

QiAccess is not responsible for:

- Storing the canonical folder hierarchy.
- Owning QiLife app data.
- Hosting runtime services.
- Defining doctrine.
