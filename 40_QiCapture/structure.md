# QiCapture Structure

QiCapture records should distinguish:

- Raw capture.
- Ingestion item.
- Extracted text or metadata.
- AI draft.
- Review decision.
- Official QiLife object.
- Timeline and activity log entry.

Every approved item should retain provenance:

- source path or source channel
- source device or agent when known
- capture timestamp
- review status
- reviewer action
- linked QiLife object ID
