# QiCapture Responsibilities

QiCapture is responsible for:

- Fast intake of raw thoughts, files, notes, messages, and documents.
- Preserving original capture text or source files.
- Creating ingestion items.
- Running extraction or inspection.
- Producing draft interpretations.
- Supporting approve, edit, or reject review actions.
- Logging approval decisions.
- Handing official records to QiLife.

QiCapture is not responsible for:

- Long-term folder taxonomy.
- Runtime hosting.
- Final app object ownership after approval.
- System doctrine.
