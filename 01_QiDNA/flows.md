# QiDNA Flows

## Documentation Placement

```text
Question
  -> identify the system layer
  -> open the matching folder
  -> read overview, responsibilities, flows, or structure
  -> use QiEOS only when the question is about doctrine or reasoning
```

## Consolidation Flow

```text
Legacy source
  -> identify useful facts
  -> remove duplicate wording
  -> resolve conflicts against the active system model
  -> place the cleaned fact in the matching system folder
```

## Authority Flow

Runtime facts beat planning notes. Active mirrored docs beat legacy documents. QiEOS explains why, but it does not duplicate every layer's operating details.
