# QiEOS Decisions

## Active Structural Decisions

- The repository is organized by live system layer, not by abstract concepts.
- `legacy_intake/` holds old source material and is not part of active navigation.
- Doctrine and reasoning live only in `01_QiDNA/QiEOS/`.
- QiServer documentation has its own layer because verified runtime facts are distinct from app and portal design notes.
- QiLife remains the app layer, while QiAccess remains the portal layer.
- QiNexus is the storage and folder discipline layer, not a generic archive.
- QiCapture is the intake pipeline and review process.
- QiConnect owns integration and access-boundary documentation.

## Conflict Rule

When old documents conflict with the active model, keep the old material in `legacy_intake/` and document only the corrected active fact in the matching layer.
