# QiDNA Responsibilities

QiDNA is responsible for:

- Maintaining the mirrored documentation structure.
- Keeping system-layer names stable.
- Holding doctrine, philosophy, decisions, and system reasoning in `QiEOS/`.
- Preventing duplicate or conflicting source-of-truth documents.
- Recording the active system model clearly and consistently.
- Separating active documentation from `legacy_intake/`.

QiDNA is not responsible for:

- Hosting runtime services.
- Replacing QiLife app documentation.
- Storing operational logs.
- Creating new architecture outside the named layers.
