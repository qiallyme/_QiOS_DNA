# QiOS DNA

QiOS DNA is the source-of-truth map for the live Qi system.

The active system model is:

```text
QiAccess -> QiLife -> (QiSystem + QiNexus + QiCapture + QiConnect)
```

QiAccess is the entry point. QiLife is the operating application. QiSystem, QiNexus, QiCapture, and QiConnect are the supporting layers around it. QiServer hosts and protects the runtime services that make those layers usable.

Use the mirrored folders to answer questions by location:

- `01_QiDNA`: system identity, doctrine, naming, and governance.
- `10_QiAccess`: portal, navigation shell, and daily entry point.
- `20_QiSystem`: operational integrity, audits, logs, generated outputs, and maintenance.
- `30_QiServer`: protected runtime host and service deployment facts.
- `40_QiCapture`: intake, ingestion, review, and approval.
- `50_QiNexus`: storage backbone and folder discipline.
- `60_QiApp_QiLife`: private life operating app and QiBit workflow.
- `70_QiConnect`: integrations, public/private access boundaries, and service connections.

Legacy source material is held in `legacy_intake/` and is not active documentation.
