# QiOS Core Doctrine

QiOS is the master operating system for the Qi ecosystem.

## Layer definitions

- **QiOS** is the whole ecosystem.
- **QiEOS** is the origin brain: doctrine, decisions, receipts, and system map.
- **QiOS Start** is the cockpit: front door, launcher, main UI, docs surface, and access dashboard.
- **QiSystem** is the rules layer: schemas, naming, metadata, database doctrine, and lifecycle rules.
- **QiServer** is the runtime and infrastructure layer.
- **QiCapture** is the ingestion and processing layer.
- **QiNexus** is the storage and workspace model.
- **QiApps** contains standalone apps, tools, and sites.
- **QiConnect** contains integrations, APIs, workers, and connector surfaces.

## Operating truth

QiOS exists to reduce drift, preserve system memory, and keep implementation repos aligned with one doctrine source.

Implementation repos can contain local docs, but they do not override QiOS DNA unless a decision record is added here.

## Promotion rule

Raw imported docs must be reviewed before becoming canonical.

```text
Raw imports are evidence.
Canonical docs are law.
Exports are snapshots.
```
