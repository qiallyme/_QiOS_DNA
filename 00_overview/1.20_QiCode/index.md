# 🜂 QiCode — Personal Legal Code
_The statutes by which I live and work._

---

## Purpose

QiCode defines the **law** — the immutable principles, statutes, and rules that govern how I operate within the QiOne ecosystem and beyond. Where QiEOS provides structure and QiKBs provide practice, QiCode provides **authority** and **commitment**.

## Principles as Law

Each Principle within a Title is a **statute** — a binding rule that shapes behavior, decision-making, and system design. Principles are:

- **Immutable by default** — Changes require formal amendment
- **Actionable** — They must guide real behavior
- **Interlinked** — Connected to workflows, SOPs, and practices
- **Traceable** — Referenced in QiKBs and client work

---

## Amendment Process

1. **Propose** a new Principle or amendment in the relevant Title directory
2. **Review** against existing Principles for conflicts
3. **Ratify** by updating the Title index and logging in `1.50_Meta/QiCodex.csv`
4. **Archive** superseded Principles in `1.90_ARCH/`

---

## QiDecimal ID Format

Each Principle follows the format: `1.XX.Y` where:
- `1.XX` = Title number (1.21 through 1.30)
- `Y` = Principle number within that Title

Example: `1.21.1` = Title I, §1 "Principle of Awareness"
